        Accessibility compliance

        Use Semantic HTML

        I have made an effort to minimize the use of div elements in my code and instead utilize appropriate semantic elements for structuring. By employing the correct elements, I aim to assist both browsers and assistive technologies like screen readers in conveying information to users effectively.

        Provide Alternative Text

        I have made an effort to include descriptive text for visual elements and avoid providing unnecessary information. Additionally, I have chosen to hide certain elements from screen readers using aria-hidden to optimize accessibility

        Optimize Color Contrast

        I utilized the built-in tools provided by the browser to inspect the color contrast and readability of text on my website. These tools include features such as filters for grayscale and blur, which helped me assess the contrast and ensure readability for users with visual impairments.

        Ensure Keyboard Accessibility / Implement Focus Styles

        I made adjustments to some elements, replacing them with the button tag to enhance user-friendliness for keyboard users and improve clarity.
